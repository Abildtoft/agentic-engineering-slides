# Junior Developers should go ALL-IN on system design

Junior developers should go ALL-IN on system design.

Historically, juniors wrote code designed by others.
Mid and senior engineers owned system design. Juniors executed.

That division made sense when writing code was the bottleneck.

It no longer is.

With AI, inference is fast and cheap.
Writing code in one stack today and another tomorrow is normal. Syntax and frameworks are becoming interchangeable. Code is increasingly disposable.

What matters more now is deciding what should exist, not just how to type it.

This raises the bar for juniors.
